«Русская цивилизация»

Город-музей из 54 виртуальных музеев о достижениях России. Патриотическое воспитание внутри страны, мягкая сила — за её пределами. Весь контент и код производит AI-агент, деньги — только на инфраструктуру.

Версия 2.0 · 20 июля 2026 · развивает архитектуру v1 от 08.07.2026 · решения Сергея от 20.07 учтены

1. Концепция и предыстория

Одной фразой: единый портал-город из 54 музеев, где каждый музей построен по доказанному стандарту, ежедневно питает соцсети контентом и ведёт аудиторию обратно на портал.

Это продолжение реального дела. В 2014 году в Переславле-Залесском открыт офлайн-комплекс «Русский парк» с музеем «Что изобрели русские первыми в мире» (~200 изобретений) — 1000+ письменных положительных отзывов. Проект закрыла экономика (аренда, кредиты, сезонность), а не контент. Виртуальный формат снимает все эти издержки и добавляет то, что офлайну недоступно: мультиязычность, интерактив, бесконечную ёмкость залов.

Что уже доказано пилотом (музей «Что изобрели русские первыми», июль 2026): календарь 365/365 дат, каталог изобретений, 11 опубликованных страниц дней с уникальными играми и hero-сценами, собственный видеодвижок (16:9 + 9:16 + постер за один прогон), скилл-стандарт страницы, факт-чекинг по русским источникам. Конвейер работает: полная страница дня со всеми форматами собирается за одну рабочую сессию агента.

2. Иерархия: четыре уровня

1
Портал — город-музей, единый вход. Творческая рисованная карта города с 3D-эффектами (решение от 20.07): павильоны-музеи стоят на «улицах»-кластерах, камера мягко парит, тап по павильону — вход. Наследует образ аллей «Русского парка».
2
Музей (54 шт.) — своя тема, свой визуальный стиль, свой скилл-файл («должностная инструкция» для агента: тон, структура, эталоны, запреты). Правило уже действует: один музей = один скилл.
3
Экспозиция — форма подачи следует теме, а не шаблону: изобретения — календарь 365 дней; художники — галерея имён и полотен; путешественники — интерактивная карта походов; военные победы — лента времени. Формы переиспользуются между музеями как движки.
4
Экспонат — страница-«день Шухова»: hero-сцена с интерактивом, статья с горькой частью, уникальная игра, квиз, видео 60–90 сек, досье AI, блок «что ещё русские сделали первыми в теме». Стартовая комплектация музея — 20–30 лучших экспонатов («первый зал»), дальше пополнение.

3. Главная страница портала — карта города-музея

Вау-момент №1 (его видит грантовая комиссия). Стилизованная карта города: холмы, река, кремль-сердце, улицы-кластеры с павильонами. Технически — canvas/three.js поверх рисованной основы: параллакс при прокрутке и наклоне телефона, дым из труб, птицы, день/ночь по местному времени, мягкое золотое свечение открытых музеев. Закрытые павильоны стоят «в лесах» — город строится на глазах, это честно и создаёт возвращаемость.

Обязательные элементы главной

  • «Сегодня в музеях» — живой пульс: сегодняшняя дата во всех открытых музеях (день изобретения, день картины…)
  • Сквозной поиск по всем экспонатам всех музеев
  • Счётчик доверия: «N экспонатов · M первенств, проверенных по источникам»
  • Языки — переключатель (этап 2, мягкая сила)
  • История проекта — «Русский парк» как предыстория, фото, отзывы
  • Донат и рекламные слоты — по стандарту v1 от 08.07.2026

5. Агентная архитектура: роли, а не штат

В отличие от ЧМЗ (16 отдельных агентов под живых сотрудников в Dify), здесь один оркестратор — агент Сергея на VPS — и 12 ролей, которые он включает как субагентов под задачу. Роль = скилл + чек-лист + выбор модели. Это дешевле, управляемее и не плодит сущности.

Девять производственных ролей (работают на каждый музей)

РольОтвечает заМодель
1. ХранительЕдиный реестр экспонатов, дубли, перекрёстные связи между музеямиSonnet

3. Факт-чекерНезависимая проверка каждого «первые в мире» по 2+ источникам. Отдельно от исследователя — принципиальноOpus
4. АвторТексты живым языком по скиллу музея: цепляющие детали, горькая частьSonnet

Три сквозные роли (одни на весь город)

  • Архитектор портала — карта города, навигация, поиск, перелинковка музеев
  • Аналитик — Яндекс.Метрика: что смотрят, где уходят, что усиливать
  • Сисадмин — бэкапы, мониторинг, доступы (ночной бэкап и watchdog уже работают)
Конвейер экспоната — путь одного факта:
Хранитель: место в реестре → Исследователь: досье с источниками → Факт-чекер: подтверждение первенства → Автор: статья + пост + сценарий → Страничник: страница + игра + hero → Видеомейкер: ролик 16:9 и 9:16 → Редактор: чек-лист, вау-контроль → публикация → соцсети

Один факт рождает связку: карточка → статья → пост → видео → игра. Это и есть экономика проекта: из одной единицы исследования — пять единиц контента.

6. Два пилота — доказательство тиражируемости

Пилот №1 · Изобретения

Работает. Календарь 365/365, 11 страниц дней, каталог по алфавиту, видеоконвейер, скилл-стандарт. Форма экспозиции — календарь. Продолжаем january-план: страницы по порядку дат.

Пилот №2 · Русское искусство

Решение от 20.07. Форма экспозиции — галерея: залы по художникам, экспонат — картина или мастер. Свой скилл (копия базового с правкой под специфику), свой стиль. Цель — показать комиссии: система переносится на любую тему, меняется форма, стандарт качества тот же.

Два музея с разными формами экспозиции + карта города = убедительная демо-модель для заявки ПФКИ: видно и качество, и масштабируемость.

Принцип: максимум статики. Статический сайт — это скорость, копеечная цена при любом трафике и минимальная поверхность атаки. Всё «живое» (игры, hero-сцены, карта города) работает в браузере посетителя без сервера.

8. Безопасность

  • Техническая: статика без БД и форм — атаковать почти нечего; HTTPS автоматом; секреты никогда не попадают в публичные репозитории; сервер — ufw, watchdog, ночные бэкапы (настроено после аудита 17.07)
  • Операционная: всё через git — любая правка откатывается; публикация только через выпускающего редактора; необратимые действия — только с подтверждения Сергея
  • Репутационная (главная для этого проекта): реестр источников на каждый факт — ни одно «первые в мире» не должно быть опровержимо без нашего готового ответа. Факт-чекер независим от исследователя; спорные первенства помечаются формулировкой точнее («первый в мире серийный…», «первый доведённый до практики…»)
